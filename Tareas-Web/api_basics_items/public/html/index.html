Endpoints Documentation

GET /items

Gets all items from the catalog.

Responses:

  • 200 OK: List of items in JSON format
  • 404 Not Found: When no items are available

GET /items/:id

Gets a specific item by its ID.

Parameters:

  • id: Numeric ID of the item

Responses:

  • 200 OK: Item in JSON format
  • 404 Not Found: When the item doesn't exist

POST /items

Registers one or more items in the catalog.

Parameters:

  • id: Numeric ID of the item (required)
  • name: Name of the item (required)
  • type: Type of the item (required)
  • effect: Effect of the item (required)

Responses:

  • 201 Created: Item(s) added successfully
  • 400 Bad Request: When attributes are missing or ID already exists

Parameters:

  • id: Numeric ID of the item to update

  • effect: New effect of the item (optional)

Responses:

  • 200 OK: Item updated successfully

POST /users

Registers one or more users with their items.

Parameters:

  • id: Numeric ID of the user (required)
  • name: Name of the user (required)
  • email: Email of the user (required)
  • items: Array of user's items (required)

Responses:

  • 201 Created: User(s) added successfully
  • 400 Bad Request: When attributes are missing, ID already exists, or some items don't exist in the catalog

Running test...

PUT /users/:id

Updates an existing user by their ID. Only updates provided fields.

Parameters:

  • id: ID of the user to update
  • name: New name of the user (optional)
  • email: New email of the user (optional)
  • items: New list of user's item IDs (optional)

Responses:

  • 200 OK: User updated successfully
  • 400 Bad Request: One or more items don't exist
  • 404 Not Found: User not found
